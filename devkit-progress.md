## 开发流程

```
PM 需求文档(含竞品调研) → Designer 设计稿 → PM+Architect 内部确认
→ Developer 开发 → Architect Code Review → QA 10轮测试 → 上线
```

## 关键决策

- 所有工具纯前端，零后端依赖
- MPA 架构，每个工具独立页面独立 SEO
- 文案常量集中管理，预留国际化
- Liquid Glass 设计规范统一
- 每个工具 10 轮 QA 测试后上线
- HEIC/TIFF 支持后续再加（MVP 先不引 heic2any）

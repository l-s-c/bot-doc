# JSON 工具网站 — 产品需求文档（PRD）

## 项目概览

**产品名称**：待定（建议：JSONKit / JSONFlow）
**目标**：打造一个设计精美、体验流畅的一站式 JSON 工具网站，超越现有竞品
**技术方案**：纯前端，HTML + CSS + JS，编辑器用 CodeMirror 6
**目标用户**：开发者、测试工程师、数据分析师

## 竞品分析

| 竞品 | 优势 | 劣势 |
|------|------|------|
| jsonformatter.org | 功能全（格式化/校验/转换/对比） | 设计过时、广告多、体验差 |
| JSON Hero | 自动识别类型（日期/URL），有个性 | 功能单一，只是 viewer |
| JSON Crack（43k star） | 设计现代，图可视化是杀手功能 | 重可视化轻工具链，高级功能付费 |

**我们的差异化：现代设计 + 一站式工具，市面上没有同时做好这两点的。**

## MVP 功能（一期）

### 1. JSON 格式化 / 美化
- 输入 JSON → 格式化输出（2/4 空格缩进可选）
- 压缩（Minify）：去掉所有空白
- 语法高亮（CodeMirror 6）
- 一键复制结果
- 支持拖拽上传 JSON 文件
- **错误处理**：非法 JSON 显示友好错误信息 + 定位到错误行

### 2. JSON 校验（Validator）
- 实时校验 JSON 合法性
- 错误信息清晰：第几行、第几列、什么错误
- 高亮错误行
- 支持宽松模式（允许尾随逗号、注释等）

### 3. 树形可视化（Tree View）
- JSON 展示为可折叠/展开的树形结构
- 显示每个节点的类型（string/number/boolean/null/array/object）
- 显示数组长度、对象属性数量
- 点击节点复制路径（如 `data.users[0].name`）
- 搜索/过滤节点
- 大文件虚拟滚动，不卡顿

### 4. JSON 对比（Diff）
- 左右双栏，输入两段 JSON
- 语义对比（不是文本 diff）：忽略 key 顺序差异
- 高亮：新增（绿）、删除（红）、修改（黄）
- 支持折叠相同部分，只看差异
- 对比结果可复制/导出

## 二期功能（后续迭代）

- JSON ↔ YAML / XML / CSV / TypeScript 类型转换
- JSONPath 查询
- JSON Schema 校验
- JSON 生成器（从 Schema 生成假数据）

## 设计要求

- **深色主题为主**，支持亮/暗切换
- **侧边栏导航**切换工具，类似 VS Code 布局
- **统一布局**：左输入 → 右输出，操作栏居中
- **微交互**：格式化动效、错误高亮闪烁、复制成功 toast
- 对标 Linear / Vercel 的设计质感
- 响应式：PC 为主，平板可用，手机基本可用
- **Logo + 品牌色**：需要 Designer 出方案

## 性能要求

- 首屏加载 < 2秒
- 10MB JSON 文件格式化 < 3秒
- 树形展开 10000+ 节点不卡顿（虚拟滚动）
- 大文件处理使用 Web Worker，不阻塞主线程

## 兼容性

- Chrome / Firefox / Edge 最新版
- 高 DPI 适配
- 最小宽度 1024px

## 空状态设计

- 输入区默认示例 JSON + placeholder "Paste JSON or drop a file"
- "Load sample JSON" 按钮一键填充
- 底部快捷键提示（⌘V / drag & drop / upload）

## Diff 补充规则

- **同步滚动**：左右栏滚动联动
- **ignoreOrder 开关**：勾选=忽略 key 顺序（语义对比），取消=key 顺序差异也标出
- **数组顺序不忽略**：`[1,2]` vs `[2,1]` 算不同
- **类型差异算修改**：`"1"` vs `1` 高亮为修改
- **变更列表面板**：展示每条变更的路径和内容

## 校验补充规则

- **宽松模式**：支持尾随逗号、单行注释（//）、多行注释（/* */）
- **不支持**：单引号、unquoted key
- **多错误标注**：尽量标出全部错误，不只第一个

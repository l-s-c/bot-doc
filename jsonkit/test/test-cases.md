## 一、JSON 格式化 & 校验（SPA）

### 格式化功能
- TC-F01: 输入合法 JSON → 格式化输出（2空格缩进）
- TC-F02: 输入合法 JSON → 格式化输出（4空格缩进）
- TC-F03: 切换缩进选项，输出实时更新
- TC-F04: 压缩（Minify）：格式化 JSON → 去掉所有空白
- TC-F05: 语法高亮正确（字符串/数字/布尔/null 不同颜色）
- TC-F06: 一键复制结果到剪贴板
- TC-F07: 拖拽上传 JSON 文件，自动填入编辑器
- TC-F08: 点击上传 JSON 文件
- TC-F09: 非法 JSON 输入 → 显示友好错误信息 + 定位到错误行
- TC-F10: 空输入 → 合理提示

### 格式化边界
- TC-F11: 大文件（1MB）格式化 < 500ms
- TC-F12: 大文件（10MB）格式化 < 3s
- TC-F13: 超大文件（50MB）格式化不崩溃
- TC-F14: 超过 50MB 文件上传 → 提示"文件过大"
- TC-F15: 深层嵌套 JSON（100+ 层）正确格式化
- TC-F16: 特殊字符：Unicode、emoji、转义字符（\n \t \" \\）
- TC-F17: 空 JSON：`{}` 和 `[]`
- TC-F18: 超长字符串值（10000+ 字符）

### 校验功能
- TC-V01: 合法 JSON → 显示"有效"
- TC-V02: 非法 JSON → 显示错误行号、列号、错误原因
- TC-V03: 错误行高亮显示
- TC-V04: 实时校验（输入时自动触发）
- TC-V05: 宽松模式 — 允许尾随逗号，校验通过
- TC-V06: 宽松模式 — 允许单行注释 `//`，校验通过
- TC-V07: 宽松模式 — 允许多行注释 `/* */`，校验通过
- TC-V08: 宽松模式 — 单引号不允许，校验失败
- TC-V09: 宽松模式 — unquoted key 不允许，校验失败
- TC-V10: 标准模式 — 尾随逗号校验失败
- TC-V11: 多个错误 → 全部标出

### 校验边界
- TC-V12: 空输入
- TC-V13: 纯空白输入
- TC-V14: 只有一个值（如 `"hello"`、`42`、`true`、`null`）— 合法 JSON
- TC-V15: 大文件（10MB）校验不卡顿

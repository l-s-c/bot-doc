# 虚拟衣橱市场调研

## 竞品分析

| 竞品 | 平台 | 特点 |
|------|------|------|
| GetWardrobe | iOS/Android/Web | 最成熟，全功能，Premium 订阅 |
| 小衣橱/衣蛙 | 国内 App | UI 一般，用户量小 |
| Cladwell | iOS | 付费 App，$3.99 买断 |
| Stylebook | iOS | 付费 App |

## 市场判断：🟢 有机会

### 值得做的理由
1. 国内没有好产品，市场有空白
2. 目标用户明确：年轻女性、穿搭博主、整理控
3. AI 差异化：拍照自动抠图+分类、搭配推荐
4. 变现路径清晰：订阅制 + 电商导流

### 技术挑战
- 需要后端：用户系统、图片存储（OSS/S3）、AI API
- 不是纯前端能做的产品
- 建议微信小程序切入（但需营业执照才能接支付）

### MVP 功能
- 拍照录入 + 自动抠图去背景
- 衣物分类管理
- 搭配推荐
- 定价：免费 50 件 + Pro ¥9.9/月不限量

## Architect 技术评估

| 功能 | 纯前端可行 | 说明 |
|------|-----------|------|
| 拍照上传 | ✅ | Camera API |
| 抠图去背景 | ⚠️ | ONNX.js + U²-Net 可行但体验不如服务端 |
| AI 分类识别 | ❌ | 需要后端/第三方 API |
| 搭配推荐 | ❌ | 需要 AI 模型 |
| 数据存储 | ⚠️ | IndexedDB 本地存，换设备丢数据 |
